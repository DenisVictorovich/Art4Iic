# КОНВЕРТЕР ПОСЛЕДОВАТЕЛЬНОГО АСИНХРОННОГО КАНАЛА В СИНХРОНИЗОВАННЫЙ ТИПА  «I2C»
Уникальность данного решения на базе примитивного микроконтроллера AVR tiny-45 заключается в том, что все конвертируемые цифровые интерфейсы построены на базе GPIO. UART и I2C здесь чисто программные. 
Когда-то наличие в достаточном количестве аппаратных интерфейсов UART или I2C в микроконтроллерах было роскошью. 
К сожалению, современные российские решения, которые в наше время (надеюсь) могут быть изготовлены средствами российских фотолитографов (пока достаточно грубый тех-процесс), содержат скромное количество аппаратных интерфейсов. 
Некоторые из них можно съэмулировать программно, что и показано в настоящем мини-проекте. Например микроконтроллер от НИИЭТ K1946BM014 (AVR) содержит лишь один UART, к тому же он выведен на PORTD, 
что затрудняет использование этого порта для других целей (например, двунаправленное управление 8-миразрядной шиной данных), 
а другие 8-миразрядные GPIO (PORTA,B,C) по разным причинам могут не подойти, либо быть необходимы для иного применения. 
Ассемблерный код переводится в машинный посредством утилиты AVRASM.EXE под DOS. Код может быть портирован на другие микроконтроллеры семейства AVR. Если кто-нибудь будет использовать это мини-решение для конвертации цифровых интерфейсов, то он может ознакомиться с мануалом ART4IIC.pdf, где описан формат транзакций с конвертером.
